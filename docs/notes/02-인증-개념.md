## NestJS vs NextAuth — 인증 방식 비교

### NestJS + Passport (처음 시도한 방식)
백엔드(NestJS)에서 인증을 처리. Controller, Service, Strategy, Guard 등 여러 파일을 만들어야 함.
→ 백엔드 지식이 필수, 코드량이 많음.

### NextAuth (Auth.js) (변경한 방식)
프론트엔드(Next.js) 안에서 인증을 처리. 설정 파일 하나로 Google 로그인 구현 가능.
→ 백엔드 없이도 동작, 코드량이 10분의 1.

### 왜 NextAuth로 바꿨는가?
- 백엔드(NestJS) 코드를 아직 이해하기 어려운 단계
- NextAuth는 Next.js만 알면 됨
- NestJS 백엔드는 DB, API가 필요해질 때 나중에 만들어도 됨

## NextAuth 전체 구조 (큰 그림)

### 사용자가 경험하는 흐름
```
1. /login 페이지 접속
2. "Google로 로그인" 버튼 클릭
3. Google 로그인 페이지로 이동 (Google 화면)
4. Google 아이디/비밀번호 입력
5. Google이 "이 사람 맞아" → 우리 사이트로 돌려보냄
6. NextAuth가 자동으로 세션(로그인 상태) 생성
7. /dashboard 페이지로 이동 (로그인 완료!)
```

### 만들어야 하는 파일들

| 파일 | 비유 | 역할 |
|---|---|---|
| `auth.ts` | 보안 시스템 설정서 | "Google 출입증을 인정한다" |
| `route.ts` | 출입구 | Google과 우리 사이트를 연결하는 통로 |
| `login/page.tsx` | 로비 | 출입증 발급 창구 (로그인 버튼) |
| `dashboard/page.tsx` | 사무실 | 출입증이 있어야 들어갈 수 있는 공간 |
| `middleware.ts` | 경비원 | "출입증 있나요?" 확인 |

### auth.ts에서 내보내는 4가지 도구

| 이름 | 역할 |
|---|---|
| `handlers` | NextAuth API 라우트 처리 (route.ts에서 사용) |
| `signIn` | 로그인 실행 함수 (로그인 버튼 클릭 시) |
| `signOut` | 로그아웃 실행 함수 |
| `auth` | 현재 로그인 상태 확인 ("이 사람 로그인했나?") |

## 사전 준비: Google Cloud Console

코드 작성 전에 Google에 "우리 사이트에서 Google 로그인을 쓸 거야"라고 신고해야 함.
→ Google Cloud Console에서 OAuth 앱 등록 → **Client ID**와 **Client Secret** 발급.

## 학습 로드맵 (NextAuth 사용 시)

백엔드 전체를 알 필요 없이, 아래 순서로 학습하면 충분:

1. **REST API 기본** — 프론트/백엔드가 데이터를 주고받는 방식 (GET, POST)
2. **OAuth 2.0 흐름** — "Google 계정으로 로그인"이 어떻게 동작하는지
3. **JWT 토큰** — 로그인 상태를 유지하는 방법 (출입증 비유)
4. **NestJS 기본** (나중에) — Module/Controller/Service 패턴
