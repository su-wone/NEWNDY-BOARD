# NB-05 · GitHub Actions CI 파이프라인 구성

> 작업일: 2026-03-18 | 상태: 완료

---

## 핵심 개념 정리

### GitHub Actions란?

GitHub에 코드를 push하거나 PR을 만들면, **자동으로 코드를 검사하고 빌드하는** 서비스다. 별도 서버를 준비할 필요 없이 GitHub가 제공하는 가상 환경(ubuntu)에서 실행된다.

### CI(Continuous Integration)란?

"지속적 통합"이라는 뜻으로, 코드를 합치기(merge) 전에 **자동으로 검사해서 문제를 미리 발견**하는 방식이다. 수동으로 매번 "빌드 되나? 타입 에러 없나?" 확인할 필요가 없어진다.

### 워크플로우 파일 구조

GitHub Actions는 `.github/workflows/` 폴더에 YAML 파일로 설정한다.

```yaml
name: CI              # 워크플로우 이름
on: ...               # 언제 실행할지 (트리거)
jobs:                 # 어떤 작업을 할지
  ci:
    runs-on: ...      # 어떤 환경에서 (ubuntu)
    steps:            # 세부 단계들
      - uses: ...     # 미리 만들어진 도구 사용
      - run: ...      # 명령어 실행
```

## CI 파이프라인 설계

요구사항에 따라 이런 흐름으로 설계했다:

```
PR 생성 또는 main에 push
  → GitHub가 가상 서버(ubuntu)를 띄움
    → Node.js 24 + pnpm 설치
      → 의존성 설치 (pnpm install)
        → lint 실행 (코드 품질 검사)
          → type-check 실행 (TypeScript 타입 검사)
            → build 실행 (빌드가 깨지지 않는지 확인)
```

하나라도 실패하면 다음 단계로 넘어가지 않고, PR에 실패 표시가 뜬다.

## 워크플로우 파일 생성

`.github/workflows/ci.yml` 파일을 만들었다. 주요 설정:

### 트리거 설정
```yaml
on:
  pull_request:
    branches: [main]    # main 대상 PR 생성/업데이트 시
  push:
    branches: [main]    # main에 직접 push 시
```

### concurrency (동시 실행 제어)
```yaml
concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true
```
같은 PR에서 새 커밋을 push하면, **이전에 실행 중이던 CI를 취소하고 새로 시작**한다. 불필요한 실행을 줄여서 시간과 비용을 절약한다.

### 의존성 설치
```yaml
- run: pnpm install --frozen-lockfile
```
`--frozen-lockfile`은 `pnpm-lock.yaml`을 **수정하지 않고 그대로 사용**하는 옵션. CI에서는 개발자가 로컬에서 확인한 것과 동일한 버전을 설치해야 하기 때문이다.

### 캐시 설정
```yaml
- uses: actions/setup-node@v6
  with:
    node-version: 24
    cache: pnpm
```
`cache: pnpm`은 의존성을 매번 새로 다운로드하지 않고 **캐시**한다. 이것이 5분 이내 목표를 달성하는 핵심이다.

## type-check 스크립트 추가

`apps/web/package.json`에 `type-check` 스크립트가 없어서 추가했다:

```json
"type-check": "tsc --noEmit"
```

- `tsc` — TypeScript 컴파일러
- `--noEmit` — JavaScript 파일을 생성하지 않고, **타입 검사만** 수행

이 스크립트가 없으면 루트에서 `pnpm type-check`를 실행해도 web 프로젝트가 건너뛰어진다.

## 로컬 검증

CI에서 실행될 세 가지 명령어를 로컬에서 테스트했다:

| 단계 | 명령어 | 결과 | 시간 |
|---|---|---|---|
| lint | `pnpm lint --filter web` | 통과 | ~2초 |
| type-check | `pnpm type-check --filter web` | 통과 | ~2초 |
| build | `pnpm build --filter web` | 통과 | ~5초 |

로컬에서 총 약 9초로, CI 환경에서도 5분 이내 목표를 충분히 달성할 수 있다.

## 머지 블록 설정 (별도 필요)

"CI 실패 시 머지 차단"은 CI 파일만으로는 설정할 수 없다. GitHub 웹에서 별도로 설정해야 한다:

**경로**: GitHub → 저장소 → Settings → Branches → Add branch protection rule
- Branch name pattern: `main`
- "Require status checks to pass before merging" 체크
- `ci` 체크를 필수로 지정

코드를 push하고 CI가 한 번 실행된 후에 설정할 수 있다.

## 티켓 요약
