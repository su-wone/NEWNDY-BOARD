### Tailwind v4에서 달라진 점

| v3 | v4 |
|---|---|
| `tailwind.config.ts` 파일로 설정 | **CSS 파일 안에서 `@theme`으로 설정** |
| `@tailwind base/components/utilities` | **`@import "tailwindcss"` 한 줄** |
| HEX/RGB 색상 | **oklch 색상** (더 넓은 색 범위) |

> 기획 문서(s1-1.md)에는 `tailwind.config.ts`로 디자인 토큰을 정의한다고 되어있지만, v4에서는 globals.css 안에서 `@theme`과 CSS 변수로 정의하는 방식으로 바뀌었다.

## shadcn/ui 초기화

```bash
cd apps/web
npx shadcn@latest init -d
```
> `-d`는 기본값(default)으로 설정한다는 옵션. 대화형 프롬프트를 건너뛴다.

### "초기화"가 하는 3가지 일

#### 1. components.json 생성

shadcn/ui의 설정 파일. 컴포넌트가 어디에 저장되고, 어떤 CSS 파일을 사용하는지 정의한다.

주요 설정:
- `rsc: true` — React Server Components 지원
- `css: "src/app/globals.css"` — 테마 변수가 적용될 CSS 파일
- `aliases.ui: "@/components/ui"` — 컴포넌트 저장 경로
- `iconLibrary: "lucide"` — 아이콘 라이브러리

#### 2. src/lib/utils.ts 생성

```ts
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

`cn()` 함수 — 여러 CSS 클래스를 합치고, Tailwind 클래스 충돌 시 나중 것이 우선되도록 병합한다.

```ts
cn("p-4 text-red", "p-2") // → "text-red p-2" (p-4가 p-2로 교체됨)
```

#### 3. globals.css에 테마 변수 추가

`--primary`, `--secondary`, `--muted`, `--accent`, `--destructive`, `--sidebar-*` 등 shadcn/ui 컴포넌트가 참조하는 모든 색상 변수가 추가됐다.

## 디자인 토큰이란?

**색상을 CSS 변수로 관리해서 한 곳에서 전체 테마를 통제하는 방식**이다.

예를 들어 Button 컴포넌트 코드를 보면:

```tsx
// button.tsx 13번 줄
variant: {
  default: "bg-primary text-primary-foreground",
  destructive: "bg-destructive/10 text-destructive",
  secondary: "bg-secondary text-secondary-foreground",
}
```

흐름을 따라가면:
```
Button에 variant="default" 사용
  → Tailwind 클래스 "bg-primary" 적용
    → Tailwind가 CSS 변수 "--primary" 값을 찾음
      → globals.css :root에서 "--primary: oklch(0.55 0.24 285)" (보라색) 적용
```

**globals.css에서 `--primary` 값만 바꾸면 모든 Button, Badge 등의 색상이 한번에 바뀐다.** 컴포넌트 코드를 하나하나 수정할 필요가 없다.

## 컴포넌트 설치

docs 요구사항에 명시된 5개 컴포넌트를 설치했다.

```bash
npx shadcn@latest add button   # 초기화 시 자동 생성됨
npx shadcn@latest add input
npx shadcn@latest add card
npx shadcn@latest add badge
npx shadcn@latest add avatar
```

> shadcn/ui는 일반 라이브러리와 다르다. `npm install`처럼 패키지를 설치하는 게 아니라, **컴포넌트 코드를 프로젝트에 직접 복사**한다. 그래서 `src/components/ui/` 안에 실제 tsx 파일이 생기고, 자유롭게 수정할 수 있다.

설치된 파일들:
```
src/components/ui/
├── avatar.tsx
├── badge.tsx
├── button.tsx
├── card.tsx
└── input.tsx
```

## 퍼플 계열 디자인 토큰 적용

globals.css의 색상 변수를 보라색 계열로 변경했다. 라이트 모드와 다크 모드 모두 적용.

| CSS 변수 | 용도 | 색상 |
|---|---|---|
| `--primary` | 메인 버튼, 강조 요소 | 보라색 |
| `--secondary` | 보조 버튼, 덜 중요한 요소 | 연한 보라 |
| `--muted` | 비활성/배경 요소 | 아주 연한 보라 |
| `--accent` | 호버 시 하이라이트 | 연한 보라 |
| `--destructive` | 삭제/위험 버튼 | 빨간색 (기본값 유지) |
| `--sidebar-*` | 사이드바 전용 색상 | 보라 계열 |

색상 형식은 `oklch(밝기 채도 색상각도)` — Tailwind v4에서 채택한 방식이다.
- 285 = 보라색 각도
- 밝기 0에 가까울수록 어둡고, 1에 가까울수록 밝다

## 설치된 의존성 정리

| `tw-animate-css` | Tailwind 애니메이션 |
