# NB-01 · npm workspace + Turborepo 모노레포 초기화

> 작업일: 2026-03-17 | 상태: 완료

---

## 핵심 개념: 모노레포란?

**모노레포(Monorepo)** = 여러 프로젝트를 하나의 Git 저장소에서 관리하는 구조.

일반적으로 프론트엔드, 백엔드를 별도의 저장소에 나누지만, 모노레포는 이렇게 한 곳에 모은다:

```
newndy-board/          ← 하나의 Git 저장소
├── apps/web/          ← 프론트엔드 (Next.js)
├── apps/api/          ← 백엔드 (NestJS)
└── packages/types/    ← 공유 코드
```

**왜 모노레포를 쓰는가?**
- 프론트/백엔드가 **같은 타입 정의**를 공유할 수 있다 (`packages/types`)
- `npm run dev` 한 번으로 **프론트/백엔드를 동시에** 실행할 수 있다
- 코드 변경 시 **영향받는 프로젝트만 자동으로 재빌드**된다 (Turborepo)

이를 가능하게 해주는 도구 조합:
- **npm workspace** — 여러 프로젝트의 의존성을 통합 관리
- **Turborepo** — 여러 프로젝트의 빌드/실행을 효율적으로 조율

---

## 사전 확인 — 개발 도구 설치 여부

모노레포를 구성하려면 Node.js, npm, Git 이 세 가지가 필수라서 먼저 확인했다.

```bash
node --version
# → v20.17.0 ✅ (nvm으로 관리)
```

```bash
npm --version
# → v11.12.0 ✅ (Node.js에 내장)
```
> npm은 Node.js를 설치하면 함께 포함되는 기본 패키지 매니저다. 별도 설치가 필요 없다.

```bash
git --version
# → git version 2.39.5 ✅ 이미 설치됨
```

**nvm으로 Node 버전 관리:**
```bash
nvm install 20.17.0     # Node 20.17.0 설치
nvm use 20.17.0         # 현재 셸에서 사용
```
> nvm(Node Version Manager)은 여러 Node.js 버전을 설치하고 프로젝트별로 전환할 수 있는 도구. `.nvmrc` 파일에 버전을 적어두면 `nvm use`만으로 자동 전환된다.

---

## 1단계 — Git 저장소 초기화

```bash
git init
```
> 이 폴더를 Git으로 관리하겠다고 선언하는 명령어. 이후 모든 코드 변경 이력을 추적할 수 있게 된다.

---

## 2단계 — package.json 생성

```bash
npm init -y
```
> 프로젝트의 기본 설정 파일(package.json)을 생성한다. 프로젝트 이름, 버전, 사용하는 라이브러리 목록 등을 담는 프로젝트의 신분증 같은 파일.

생성된 기본 package.json을 모노레포에 맞게 수정했다.

**왜 수정했나?**
- `name`: `Newndy_Board` → `newndy-board` — npm 패키지 이름은 소문자+하이픈 규칙을 따라야 해서
- `private: true` 추가 — "이 루트는 직접 배포하지 않고, 하위 프로젝트들을 관리만 한다"는 선언
- `main: "index.js"` 삭제 — 모노레포 루트에는 진입 파일이 필요 없어서
- `scripts`에 `dev`, `build`, `lint`, `type-check` 추가 — Turborepo가 하위 프로젝트들의 명령어를 한번에 실행하도록
- `workspaces` 추가 — npm에게 워크스페이스 범위를 알려주기 위해

---

## 3단계 — npm 워크스페이스 설정

package.json에 `workspaces` 필드를 추가했다.

```json
"workspaces": ["apps/*", "packages/*"]
```
> "이 모노레포에서 어떤 폴더들이 각각의 프로젝트인지" npm에게 알려주는 설정.
> `apps/*`는 apps 폴더 안의 모든 프로젝트(web, api), `packages/*`는 packages 폴더 안의 공유 패키지(types)를 의미.
> npm은 별도 파일 없이 package.json 안에서 워크스페이스를 정의한다.

---

## 4단계 — 폴더 구조 생성

모노레포의 실제 폴더 구조를 만들었다. `mkdir -p`는 "중간 폴더가 없으면 자동으로 함께 생성"하는 옵션.

```bash
mkdir -p apps/web        # 프론트엔드 (Next.js)
mkdir -p apps/api        # 백엔드 (NestJS)
mkdir -p packages/types  # 프론트/백엔드가 공유하는 TypeScript 타입 정의
```

---

## 5단계 — Turborepo 설치

```bash
npm install -D turbo
```
> Turborepo는 모노레포 안의 여러 프로젝트를 효율적으로 빌드/실행해주는 도구. 예를 들어 `turbo dev`를 실행하면 web과 api를 동시에 시작해준다.
> - `-D`: 개발용 의존성으로 설치. 배포할 때는 포함되지 않고, 개발할 때만 사용됨

---

## 6단계 — turbo.json 설정

Turborepo의 설정 파일을 만들었다. "어떤 명령어를 어떤 순서로 실행할지" 정의하는 파일.

- `build` — 프로덕션 빌드. `dependsOn: ["^build"]`는 의존하는 패키지(types)를 먼저 빌드한 후 실행
- `outputs` — `.next/**`(Next.js), `dist/**`(NestJS) 빌드 결과물을 캐싱 대상으로 지정
- `dev` — 개발 서버. `cache: false`(캐싱 불필요), `persistent: true`(계속 실행 상태 유지)
- `lint` — 코드 검사. 빌드 후 실행해야 타입 정보를 참조 가능
- `type-check` — TypeScript 타입 검사. 마찬가지로 빌드 의존

---

## 7단계 — .gitignore 생성

Git이 추적하지 않을 파일 목록을 정의했다.

- `node_modules/` — 설치된 라이브러리 (용량이 크고 npm install로 재생성 가능)
- `.env` — 비밀 키가 포함된 환경 변수 파일 (유출 방지)
- `.next/`, `dist/` — 빌드 결과물 (빌드 명령어로 재생성 가능)
- `.turbo/` — Turborepo 캐시
- `.DS_Store` — macOS가 자동 생성하는 파일

---

## 8단계 — .nvmrc 생성

프로젝트에서 사용할 Node.js 버전을 고정하는 파일을 만들었다.

```
20.17.0
```
> 팀원이 프로젝트 폴더에서 `nvm use`를 실행하면 자동으로 Node 20.17.0을 사용하게 된다. 팀 전체가 같은 Node 버전을 쓰도록 강제하는 역할.

---

## 9단계 — packages/types 초기화

공유 타입 패키지를 초기화했다. 이 패키지는 프론트엔드(web)와 백엔드(api)가 공통으로 사용하는 TypeScript 타입을 모아두는 곳.

```bash
mkdir -p packages/types/src
```

package.json에서 `@newndy-board/types`라는 이름으로 만들면, 다른 프로젝트에서 이렇게 가져다 쓸 수 있다:
```ts
import { User } from "@newndy-board/types"
```

`src/index.ts`는 이 패키지의 진입 파일. 지금은 비어있지만 나중에 User, Task 같은 공유 타입을 여기에 정의하게 된다.

## 최종 구조

```
newndy-board/
├── apps/
│   ├── web/              ← Next.js (NB-02에서 생성 예정)
│   └── api/              ← NestJS (추후 생성 예정)
├── packages/
│   └── types/            ← 공유 타입 패키지 (초기화 완료)
│       ├── package.json
│       └── src/index.ts
├── .gitignore
├── .nvmrc                ← Node 버전 고정 (20.17.0)
├── package.json          ← 모노레포 루트 설정 (workspaces 포함)
├── package-lock.json     ← 의존성 잠금 파일 (자동 생성)
└── turbo.json            ← Turborepo 설정
```
