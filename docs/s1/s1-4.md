## 배경

Newndy Board는 팀장(Admin)과 팀원(Member)의 역할이 명확히 구분됩니다. AI Task 생성 및 배치 결정은 팀장 전용 기능이며, 이후 모든 권한 분기 로직의 기반이 되므로 초기에 구조를 잡아야 합니다.

---

## 방향성

DB 레벨의 역할(Role) 모델과 NestJS Guard를 통해 API 접근을 제어하고, 프론트엔드에서도 역할에 따라 UI를 분기합니다. 역할은 Admin / Member 두 가지로 단순하게 유지합니다.

---

## 제품 요구사항

### 1. 역할 DB 모델
- `WorkspaceMember` 테이블에 `role: ADMIN | MEMBER` 컬럼 추가
- 워크스페이스 생성자는 자동 ADMIN, 초대된 팀원은 기본 MEMBER

### 2. NestJS Role Guard
- `@Roles('ADMIN')` 데코레이터 기반 Guard 구현
- Admin 전용 API 엔드포인트 보호 (AI Task 생성, 스프린트 시작/완료 등)
- 권한 없는 요청 시 403 응답

### 3. 프론트엔드 UI 분기
- 로그인 유저의 역할 정보를 전역 상태로 관리
- Admin 전용 메뉴 / 버튼 조건부 렌더링 (AI 생성 버튼, 팀원 관리 등)
