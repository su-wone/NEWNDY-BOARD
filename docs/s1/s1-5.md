## 배경

이후 모든 기능 페이지의 진입점이 되는 사이드바 레이아웃과 6개 라우팅을 먼저 잡아야 합니다. 스페이스 페이지는 Git Project 페이지로 교체되었으며, 이를 반영한 라우팅 구조를 초기에 확정합니다.

---

## 방향성

Next.js App Router의 Layout 시스템을 활용해 사이드바를 공통 레이아웃으로 구성합니다. 각 페이지는 플레이스홀더 UI로 시작하며, 이후 Epic 2~6에서 실제 콘텐츠로 채워집니다.

---

## 제품 요구사항

### 1. 사이드바 레이아웃 컴포넌트
- 로고 영역, 네비게이션 메뉴, 하단 유저 프로필 영역 구성
- 현재 활성 페이지 하이라이트 처리
- 반응형 처리 (모바일 시 접힘)

### 2. 6개 페이지 라우팅 구성

| 경로 | 페이지명 | 비고 |
|---|---|---|
| `/dashboard` | 요약 (대시보드) | AI Task 생성 포함 예정 |
| `/backlog` | 백로그 | Epic 2에서 구현 |
| `/sprint` | 활성 스프린트 | Epic 2에서 구현 |
| `/git` | Git Project | Epic 6에서 구현 (신규) |
| `/team` | 팀 | Epic 5에서 구현 |
| `/settings` | 설정 | Epic 5에서 구현 |

### 3. 플레이스홀더 UI
- 각 페이지에 페이지명과 "준비 중" 텍스트 표시
- 공통 페이지 타이틀 컴포넌트 적용
