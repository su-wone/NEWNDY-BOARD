## 배경

Newndy Board 전체 개발에 앞서 프론트엔드 기반 환경과 자동화 파이프라인을 먼저 구축해야 합니다. 이 작업이 완료되지 않으면 이후 모든 Sprint의 기능 개발, 테스트, 배포가 불가능하므로 Sprint 1의 첫 번째 Story로 편성합니다.

---

## 방향성

Next.js 16 (App Router) + Tailwind CSS 기반 프론트엔드 프로젝트를 초기 구성하고, GitHub Actions CI/CD 파이프라인을 자동화합니다. 팀 전체가 동일한 개발 환경에서 작업할 수 있는 표준화된 기반을 만드는 것이 목표입니다.

---

## 제품 요구사항

### 1. 모노레포 구조 셋업
- pnpm workspace + Turborepo 기반 초기화
- `/apps/web` (Next.js), `/apps/api` (NestJS), `/packages/types` (공유 타입) 폴더 구성

### 2. Next.js 16 초기 설정
- App Router 기반, TypeScript `strict: true`
- 6개 페이지 라우팅 플레이스홀더 (`/dashboard`, `/backlog`, `/sprint`, `/git`, `/team`, `/settings`)
- 로그인 여부에 따른 미들웨어 리다이렉트 처리

### 3. Tailwind CSS 설정
- `tailwind.config.ts` 디자인 토큰 정의 (색상·타이포그래피·간격)
- shadcn/ui 초기 설치 (Button, Input, Card, Badge, Avatar)

### 4. 코드 품질 도구
- ESLint + Prettier + Husky lint-staged 설정
- `.env.example` 작성 (팀원 온보딩용)

### 5. GitHub Actions CI 파이프라인
- PR 생성 시 자동 트리거: `lint` → `type-check` → `build` 순 실행
- 실패 시 머지 블록, 전체 실행 5분 이내 목표

### 6. Vercel 자동 배포 연동
- `main` 머지 시 프로덕션 자동 배포
- PR 생성 시 Preview URL 자동 생성 및 PR 코멘트 첨부
